## 1. Общая процедура для КАЖДОГО из 7 типов
Повтори шаги 1–7 для каждого типа из таблицы ниже.

1. Открой https://messenginfo.com/ru/services/translator
2. Нажми «Начать перевод →».
3. На экране «Какой документ нужно перевести?» выбери нужную плитку.
4. Загрузи фото документа (JPEG/PNG/HEIC, до 10 МБ).
5. Дождись извлечения полей — обычно **20–60 секунд**. Не перезагружай страницу.
6. Проверь таблицу распознанных полей (строки `.tw-trans-row`):
   - **≥3 поля заполнены реальными значениями** — НЕ «—» и НЕ «Не найдено».
   - На рукописных полях стоит бейдж **✍️** (review) — такие поля система НЕ
     финализирует сама, они требуют твоего подтверждения. Это норма, не баг.
   - Подтверди (confirm) хотя бы одно ✍️-поле — значение сохраняется, бейдж
     снимается / поле помечается как проверенное.
7. Перейди дальше до предпросмотра PDF — **PDF-превью рендерится** (не пустая
   страница, не ошибка), поля из таблицы видны в переводе.

## 2. HEIC-тест (iPhone)
1. Сфотографируй любой из документов на iPhone (формат по умолчанию = HEIC) и
   загрузи в визард **прямо с iPhone Safari**, без конвертации.
2. Ожидаем: загрузка принята (НЕТ ошибки 415 «unsupported media type»), поля
   извлекаются как обычно. Контрольная точка: на prod-пробе HEIC дал **5/5 полей**.
3. **Известное ограничение:** HEIC **>3.8 MB на десктопном Chrome** может дать
   413 — браузер не умеет ужать HEVC на клиенте. Типичные iPhone HEIC 1.5–3 MB
   проходят. Поэтому HEIC тестируем с iPhone Safari, не с десктопа.

## 3. Тест поворота (rotation)
1. Сфотографируй ОДИН документ боком (90°) и/или вверх ногами и загрузи.
2. Ожидаем: авто-ориентация срабатывает, поля всё равно извлекаются (≥3 поля
   со значениями). Если все поля пустые на повёрнутом фото, но заполняются на
   прямом — это провал теста, фиксируй в отчёте.
