## Mobile 375px — Tailwind CSS Analysis (Programmatic)

Visual screenshots at 375px were not possible (browser minimum 720px, sandbox couldn't install Puppeteer). Instead, a full programmatic audit of Tailwind responsive classes was performed — this verifies the CSS rules that WOULD apply at 375px.

**Framework:** Next.js + Tailwind CSS
**Breakpoints found:** sm (640px), md (768px), lg (1024px), custom min-[600px]

### Header at 375px:
- Main navigation (4 items): `hidden md:flex` → **HIDDEN** below 768px ✅
- "Sign in" button: `hidden sm:inline-flex` → **HIDDEN** below 640px ✅
- "Check Status" button: `hidden sm:inline-flex` → **HIDDEN** below 640px ✅
- Visible at 375px: logo + theme toggle + language switcher only ✅

### Content at 375px:
- Service cards: `grid grid-cols-1 sm:grid-cols-2` → **SINGLE COLUMN** ✅
- Quick links: `grid grid-cols-1 lg:grid-cols-4` → **SINGLE COLUMN** ✅
- Card grid: `grid grid-cols-1 min-[600px]:grid-cols-2` → **SINGLE COLUMN** below 600px ✅
- Footer: `grid grid-cols-1 md:grid-cols-4` → **STACKED** ✅
- Bottom nav: `grid grid-cols-4` (always) → 4 equal tabs ✅

### Potential issues at 375px:
- Chat widget covers ~15% of viewport width (vs ~6% at desktop)
- Hero badge + info card layout needs visual verification (no responsive classes found for hero flex direction)
