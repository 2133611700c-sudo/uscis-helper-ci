# Re-Parole — Service Flow Inputs

Updated: 2026-04-30 (Phase 1C pilot evidence from YT-IMIGRANT)

## 1. User problem
- Users ask whether travel/re-entry is safe under parole and what duration/conditions apply.
- Users ask how Re-Parole interacts with status risks after negative case events.

## 2. Common user questions from sources
- Can advance parole be valid for up to 5 years?
- Can I leave and return safely with current documents?
- What is my status risk after denial-related scenarios?

## 3. Documents/data users ask about
- Passport
- I-94
- Re-Parole/Advance parole document details
- Receipt number and case-status output

## 4. Fields users are confused about
- Travel authorization validity period
- Which status controls after denial events
- What combination of docs is required for travel context

## 5. What the service should generate
- Re-Parole checklist by user scenario
- Travel-risk warning state (not legal advice)
- Official-source links grouped by scenario

## 6. What must remain user responsibility
- Legal strategy decisions
- Final interpretation of travel risk
- Confirming all dates and status details before travel

## 7. Official sources needed
- USCIS I-131: https://www.uscis.gov/i-131
- USCIS Case Status: https://egov.uscis.gov/
- DOJ EOIR roster: https://www.justice.gov/eoir/recognized-organizations-and-accredited-representatives-roster

## 8. High-risk routing rules
- Auto-route to attorney/DOJ for:
  - denial-linked status exposure
  - travel outside US with unclear status
  - contradictory evidence/status records

## 9. Bot answer opportunities
- “What docs do I need for Re-Parole travel context?”
- “Can I travel now under my current status?”

## 10. Open verification items
- Verify any fixed-duration travel claims from creator content against USCIS pages.
- Add official date-stamped references for all duration-dependent statements.
