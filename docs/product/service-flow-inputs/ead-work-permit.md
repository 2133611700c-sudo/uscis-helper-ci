# EAD / Work Permit — Service Flow Inputs

Updated: 2026-04-30 (Phase 1C pilot evidence from YT-IMIGRANT)

## 1. User problem
- Users ask what changed in EAD filing/renewal and what they should do now.

## 2. Common user questions from sources
- What changed for EAD in 2026?
- Which filing path is now correct for my case?
- How to avoid rejection due to outdated assumptions?

## 3. Documents/data users ask about
- I-765 context
- Current case status/receipt info
- Existing EAD data (if any)
- Identity/supporting documents used for work authorization flow

## 4. Fields users are confused about
- Renewal vs initial decision path
- Which supporting evidence set is required by scenario
- Filing channel assumptions (online vs other)

## 5. What the service should generate
- Scenario-based EAD checklist
- “What changed / what to verify” block
- Official-source link package for each scenario

## 6. What must remain user responsibility
- Final form data correctness
- Filing method choice confirmation
- Decision on high-risk edge cases

## 7. Official sources needed
- USCIS I-765: https://www.uscis.gov/i-765
- USCIS All Forms: https://www.uscis.gov/forms/all-forms

## 8. High-risk routing rules
- Route to attorney for:
  - denial/RFE history tied to work authorization
  - conflict between creator advice and official source
  - status uncertainty affecting eligibility

## 9. Bot answer opportunities
- “What changed in EAD rules?”
- “What should I prepare before filing I-765?”

## 10. Open verification items
- Validate all 2026-change claims against USCIS updates with last checked date.
- Map scenario differences only after official citation is attached.
